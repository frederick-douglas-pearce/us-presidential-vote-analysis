## 2. `ARCHITECT_TRIGGERS`

Fire `DESIGN_AGENT` (`architect`) when a plan touches any of the following; bias toward calling it.

- **Shared/cross-source contracts:** anything under `usvote/pv/` (the `schema.py` record shape,
  `status.py` roster/enum, `pv/load.py` write seams), or the `dwh` DDL / star-schema grain.
- **Source-namespacing boundary (D006/D015/D027):** a new top-level EC-domain module, a new PV
  subpackage stage, or any change to the composition root (`usvote/warehouse.py`) or the
  `spine.py`/`years.py`/`join.py` family. The greppable invariant (nothing under `usvote/{mit,ucsb,pv}/`
  imports `usvote.warehouse`; nothing under `usvote/pv/` names `dwh.votes`) must not regress.
- **The EC↔PV join seam (D017/D026):** changes to `join.py`, the `ec_pv_preferred` /
  `ec_pv_redistributable` views, or the EC-left / dense-fact assumptions.
- **The API serving boundary (D028/D031):** the snapshot contract (`snapshot_schema.py`,
  `docs/api-snapshot.md`), the `usvote/api/` import graph ("no live DB at serve time"), or any new
  `/v1` endpoint / response envelope.
- **A new historical data correction or PV source**, or when the orchestrator is unsure.
- **Skip** for docs-only, comment/typo, and pure test-addition changes.

Any decision worth recording lands as a new `## D0NN` entry in `.claude/specs/decisions.md`
(append-only — see the guard).

## 3. `SOURCE_LAYOUT` — router signals

- **Package layout:** source in `src/usvote/` (installable package); tests in `tests/unit/`
  (offline) and `tests/integration/` (live-Postgres, `@pytest.mark.integration` — the *marker*,
  not the dir, selects them); fixtures in `tests/fixtures/`. Legacy notebooks (`step*.ipynb`) are
  retained during migration and **excluded** from ruff/the package lint policy — not the pipeline
  spine. No runtime-dep leakage into tests.
- **`docs` label / path:** `documentation` label; docs confined to `docs/` (+ `README.md`,
  `CLAUDE.md`). Decision log at `.claude/specs/decisions.md`.
- **`research` label / path:** `research` label — research-spike / written-recommendation
  deliverable (e.g. #70 PV-discrepancy quantification); no test-coverage gate.
- **`stub-defer` marker:** exclude from selection any issue whose **title begins with `Epic:`** —
  these are umbrella trackers (#94/#63/#33/#12), not directly implementable. No real work issue
  uses that title prefix, so the rule is greppable with no extra labels.
- **`code` (default):** bug/enhancement touching `src/usvote/` → full pipeline (lint + mypy +
  pytest, integration when the DB write path changes).

## 4. Security routing

> All rules here are GitHub/host-specific. Host **is** GitHub. No dedicated security workflow
> exists yet (only `.github/workflows/ci.yml`) — routing is via the built-in `/security-review`.

- **`.claude/`-only change** → run local `/security-review` (no CI security workflow to trigger).
- **Sensitive surface** — the API serve layer (`usvote/api/`), the DB write/connection path
  (`usvote/db.py`, `usvote/load.py`, `usvote/pv/load.py`), or scraping/network code
  (`usvote/scrape.py`, `usvote/ucsb/` network stage) → run `/security-review` when dev-complete.
  Note the repo's own guardrails: no UCSB bytes ever committed (public repo, D022), snapshot is
  redistributable-only at the source (D030), and the wheel uses `packages=` not `force-include`
  (avoids bundling secrets — see `pyproject.toml`).
- **Skip** for docs / no-surface changes.
- **Decided: security review stays purely local** via `/security-review`. No `needs-security-review`
  label or labeled GitHub workflow — this repo has no CI security job and no API-key/billing setup to
  support one; local review covers the sensitive surfaces above when a branch is dev-complete.
